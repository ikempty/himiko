# 日本のチャットアプリ風 会話画面ジェネレーター

昔話や童話を、主人公目線のグループチャット会話として再構成するための静的HTML/CSS/JavaScriptテンプレートです。  
実在アプリの完全再現は避けつつ、日本人が見てすぐにチャット画面と分かる雰囲気を狙っています。

## 起動方法

1. このフォルダの `index.html` をブラウザで開きます。
2. ビルドや npm は不要です。
3. ローカルサーバーで開いても構いません。

## ファイル構成

- `index.html`
  画面の骨組みです。操作パネルと縦型プレビューを配置しています。
- `styles.css`
  レイアウト、吹き出し、配色、スマホ風プレビューの見た目を定義します。
- `app.js`
  投稿切替、描画、クリーン表示、1件ずつ増える自動再生、速度切替を担当します。
- `story-data.js`
  作品データです。別作品に差し替えるときは基本的にこのファイルだけ編集します。
- `README.md`
  使い方とデータ構造の説明です。

## できること

- 9:16 の縦型プレビュー表示
- 投稿 1 / 投稿 2 / 投稿 3 の切替
- 主人公の右吹き出し / 他キャラの左吹き出し表示
- `message` / `system` / `date` の3種類の表示
- 通常プレビューとキャプチャ向けクリーン表示の切替
- 1メッセージずつリアルタイムに増えていく自動再生
- 自動再生の速度切替
- キーボード操作

## キーボード操作

- `←` `→`
  前後の投稿へ移動します。
- `1` `2` `3`
  投稿 1 / 投稿 2 / 投稿 3 に切り替えます。
- `P`
  自動再生を開始 / 停止します。
- `C`
  クリーン表示を切り替えます。
- `Esc`
  クリーン表示を解除します。

## `story-data.js` の編集方法

このアプリは `story-data.js` の `STORY_DATA` オブジェクトを読み込んで表示します。  
作品差し替え時は、基本的に `story-data.js` だけを編集する想定です。

基本構造は次のとおりです。

```js
const STORY_DATA = {
  workId: "momotaro",
  workTitle: "桃太郎",
  theme: {
    accentColor: "#79dd67",
    backgroundColor: "#2d2f30"
  },
  posts: [
    {
      postIndex: 1,
      groupName: "川で拾った桃の件",
      participants: ["おばあ", "おじい", "桃太郎"],
      messages: [
        { type: "system", text: "位置情報: 川辺ルートが共有されました" },
        {
          type: "message",
          sender: "おばあ",
          text: "ちょっと川行ってくる",
          isSelf: false,
          time: "08:12",
          avatarText: "お"
        }
      ]
    }
  ]
};
```

## 投稿とメッセージの定義方法

### 投稿

各投稿は `posts` 配列の1要素です。

- `postIndex`
  投稿番号です。表示順に使います。
- `groupName`
  画面上部に表示するグループ名です。
- `participants`
  その投稿に登場する人物一覧です。
- `messages`
  表示する会話配列です。

### メッセージ

`messages` の各要素は次のフィールドを使えます。

- `type`
  `"message"` / `"system"` / `"date"` を指定します。
- `sender`
  発言者名です。`type: "message"` のときに使います。
- `text`
  表示本文です。改行も使えます。
- `isSelf`
  `true` なら主人公として右側吹き出し、`false` なら左側吹き出しです。
- `time`
  吹き出し下に表示する時刻です。
- `emphasis`
  `true` にすると少し強調表示されます。
- `avatarText`
  左側メッセージの簡易アイコン用1〜2文字です。
- `note`
  将来の拡張用に保持できます。現在の標準UIでは画面表示しません。

## 新しい作品を追加する方法

1. `story-data.js` の `STORY_DATA` を複製します。
2. 次の差分だけを書き換えます。
   - `workId`
   - `workTitle`
   - `theme`
   - 各投稿の `groupName`
   - `participants`
   - `messages`
3. `index.html` を開き直して確認します。

## キャプチャモードの使い方

- `クリーン表示` ボタン、または `C` キーで切り替えます。
- クリーン表示では操作パネルを隠し、縦型プレビューだけを中央に表示します。
- `Esc` で解除できます。

## 自動再生の使い方

- `自動再生` ボタン、または `P` キーで開始 / 停止します。
- 再生中は、メッセージが1件ずつリアルタイムで追加されます。
- 投稿の最後まで行くと次の投稿へ進み、最後の投稿まで再生すると停止します。
- `再生速度` で表示テンポを切り替えられます。
